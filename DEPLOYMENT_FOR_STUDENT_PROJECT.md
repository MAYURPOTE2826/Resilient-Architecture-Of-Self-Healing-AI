### **OPTION 2: Render.com (ALSO EASY)**

**Best for:** Alternative to Railway

**Setup time:** 20 minutes  
**Cost:** Free (limited to 512 MB RAM)  
**Includes:** Managed PostgreSQL

#### Step-by-Step:

```bash
# 1. Create Render account
→ Visit: https://render.com
→ Sign up with GitHub

# 2. Create new service
→ Dashboard → New +
→ Select: Web Service
→ Connect GitHub repo

# 3. Configure
→ Runtime: Python 3.11
→ Build command: pip install -r requirements.txt
→ Start command: gunicorn src.api:app

# 4. Environment variables
→ Add from .env template:
   SECRET_KEY=<generate>
   JWT_SECRET=<generate>
   DATABASE_URL=<config in Render>

# 5. Deploy
→ Click: Create Web Service
→ Wait: 5-10 minutes
→ Your app is live: https://your-app.onrender.com
```

**Limitations:**
- Free tier spins down after 15 minutes of inactivity (wakes up when visited)
- 512 MB RAM (sufficient for demo)
- Limited to small concurrent users

**Best for:** Occasional demos during project evaluation

### **OPTION 3: Google Cloud Run (ADVANCED)**

**Best for:** If you want to learn cloud platforms

**Setup time:** 30 minutes  
**Cost:** Free (12-month free trial, $90/month credit)  
**Includes:** Serverless hosting, CloudSQL

#### Step-by-Step:

```bash
# 1. Create Google Cloud account
→ Visit: https://console.cloud.google.com
→ Enable free trial ($90 credit)

# 2. Create Cloud SQL instance
→ SQL → Create Instance
→ PostgreSQL 15
→ Single zone (cheaper)
→ 10 GB storage

# 3. Deploy to Cloud Run
→ gcloud auth login
→ gcloud config set project <project-id>
→ gcloud run deploy self-healing \
    --source . \
    --platform managed \
    --region us-central1 \
    --allow-unauthenticated

# 4. Environment variables
→ Cloud Run → Edit service
→ Set environment variables from .env

# 5. Access
→ Your app is at: https://self-healing-xxxxx.a.run.app
```

**Advantages:**
- Google Cloud (resume builder)
- Generous free credits ($90/month for 12 months)
- Auto-scaling
- Professional infrastructure

## 📋 FULL DEPLOYMENT STEPS (Railway - Easiest)

### **STEP 1: Prepare GitHub (5 minutes)**

```bash
# Make sure everything is committed
git status
# Should be: nothing to commit, working tree clean

# If not clean:
git add .
git commit -m "Final project version for deployment"
git push origin main
```

### **STEP 2: Create Railway Account (2 minutes)**

```
1. Go to: https://railway.app
2. Click: "Start Building"
3. Click: "Deploy with GitHub"
4. Authorize Railway to access your repos
5. Done!
```

### **STEP 3: Create New Project (3 minutes)**

```
In Railway Dashboard:
1. Click: "New Project"
2. Select: "Deploy from GitHub repo"
3. Find: "self-healing-engine" repo
4. Click: "Deploy Now"
```

### **STEP 4: Configure Environment (5 minutes)**

```
In Railway:
1. View Deployments
2. Click on deployment
3. Click: "Variables"
4. Click: "New Variable"
5. Add these variables:

   FLASK_ENV=production
   SECRET_KEY=<run: python3 -c "import secrets; print(secrets.token_hex(32))">
   JWT_SECRET=<run: python3 -c "import secrets; print(secrets.token_hex(32))">
   LOG_LEVEL=INFO
   CORS_ORIGINS=*
   
6. Save
```

### **STEP 5: Deploy (Automatic or Manual)**

```
Railway will:
1. Detect your Dockerfile
2. Detect docker-compose.yml
3. Build automatically
4. Deploy automatically
5. Show your live URL

Wait for: "✓ Build and deploy successful"
```

### **STEP 6: Verify Live (2 minutes)**

```bash
# Your app URL will be something like:
https://self-healing-engine-prod.railway.app

{"status": "healthy", ...}
```
